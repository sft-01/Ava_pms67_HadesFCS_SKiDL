# Ava_pms67_HadesFCS_SKiDL

SKiDL-based schematic scripts for the **Hades Flight Control System** by Philip M. Salmony — reimplemented programmatically using [SKiDL](https://devbisme.github.io/skidl/), a Python library for defining electronic circuits as code.

---

## What is SKiDL?

SKiDL lets you describe hardware schematics in pure Python. Instead of drawing circuits in a GUI, you define components, nets, and connections in code — making designs version-controllable, testable, and reusable like software.

---

## Repository Structure

Each board has its own set of files:

| File | Description |
|------|-------------|
| `*_skidl.py` | SKiDL script defining all components, nets, and connections |
| `*_skidl.net` | KiCad netlist generated by running the SKiDL script |
| `*_skidl_sklib.py` | SKiDL library file |
| `*_skidl.erc` | Electrical rules check output |
| `*_skidl.log` | Run log |
| `*_original.net` | Reference netlist from the original KiCad schematic |

### Boards included

- **HadesGPSDongle** — GPS dongle board (27 components, 27 nets)
- **HadesMicro** — Micro flight controller board (87 components, 161 nets)
- **HadesMicroJLCPCB** — JLCPCB-optimized variant of HadesMicro (84 components, 175 nets)
- **HadesNT** — Navigation and telemetry board (102 components, 144 nets)

## How to Run

### Requirements

```bash
pip install skidl
```

### Generate a netlist

```bash
python3 HadesGPSDongle_skidl.py        # outputs HadesGPSDongle_skidl.net
python3 HadesMicro_skidl.py            # outputs HadesMicro_skidl.net
python3 HadesMicroJLCPCB_skidl.py      # outputs HadesMicroJLCPCB_skidl.net
python3 HadesNT_skidl.py               # outputs HadesNT_skidl.net
```

### Verify against reference

Use the included comparison script to confirm the generated netlist is correct:

```bash
python3 compare_netlists.py HadesGPSDongle_original.net HadesGPSDongle_skidl.net
```

A passing result looks like:

```
============================================================
  🎉  PERFECT MATCH — 100% identical netlists!
============================================================
```

The script checks:
- Component count, references, values, and footprints
- All named and unnamed net names
- Every pin-to-net connection

## Design Approach

Each SKiDL script follows the same structure:

1. **Nets** — declared with `Net.fetch()` for named nets (prevents SKiDL from auto-suffixing shared power nets like `GND` → `GND1`, `GND2`...) and `Net()` for unconnected pins
2. **Components** — instantiated with explicit pin definitions using `Part(tool=SKIDL, ...)`
3. **Connections** — each pin explicitly wired to its net via `+=`
